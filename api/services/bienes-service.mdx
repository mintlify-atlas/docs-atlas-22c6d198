## Overview

**Package:** `com.bienes.services`

**Full Class Name:** `com.bienes.services.BienesServices`

The `BienesServices` class provides business logic for managing goods and assets in the system. It acts as an intermediary between the presentation layer (servlets) and the data access layer (DAOs), implementing CRUD operations for the `Bienes` entity.

### Dependencies

- **DAO Factory:** `DAOFactory.getDAOFactory(1)` - MySQL implementation
- **DAO Interface:** `BienesDAO`
- **Entity:** `com.bienes.entidad.Bienes`

```java BienesServices.java:11-14
private DAOFactory fabrica = DAOFactory.getDAOFactory(1);
```

## Error Handling

The service layer returns integer values for CUD operations (Create, Update, Delete):

- **Success:** Return value `> 0` indicates the number of affected rows
- **Failure:** Return value `<= 0` indicates operation failed

### Common Error Scenarios

1. **Invalid Purchase Order Code:** When registering a Bien with a non-existent `codigoOrdendeCompra`, the operation fails with return value `0`

2. **Entity Not Found:** When updating or deleting a non-existent Bien, the operation returns `0`

3. **Database Constraints:** Foreign key violations or other database constraints will cause operations to fail

### Servlet-Level Error Handling

Servlets check return values and set appropriate messages:

```java
if (salida > 0) {
    // Success message
    request.setAttribute("MENSAJE", "Operation successful");
} else {
    // Error message
    request.setAttribute("MENSAJE", "Operation failed");
}
```
