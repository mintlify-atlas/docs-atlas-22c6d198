## Overview

**Package:** `com.bienes.services`

**Full Class Name:** `com.bienes.services.GuiaServices`

The `GuiaServices` class provides business logic for managing purchase order guides (Gu├şa y Orden de Compra) in the system. This service handles the lifecycle of purchase orders that are referenced by Bienes records.

- **DAO Factory:** `DAOFactory.getDAOFactory(1)` - MySQL implementation

## Usage Examples

### Service Initialization

Typical servlet initialization pattern:

```java
private GuiaServices servicio;

public ServletGuia() {
    super();
    servicio = new GuiaServices();
}
```

### Registering a New Purchase Order

```java
GuiayOrdendeCompra bean = new GuiayOrdendeCompra();
bean.setNom_provee("Provider Name");
bean.setFecha_compra(Date.valueOf("2024-03-15"));
bean.setDescripcionguiacompra("Office supplies");
bean.setCantidadguiacompra("50");
bean.setPrecioguiacompra(1500.00);

int salida = servicio.registrar(bean);
if (salida > 0) {
    // Success - purchase order created
    System.out.println("Purchase order registered successfully");
} else {
    // Error handling
    System.out.println("Failed to register purchase order");
}
```

### Updating a Purchase Order

```java
GuiayOrdendeCompra bean = servicio.buscarPorId(orderCode);
if (bean != null) {
    bean.setPrecioguiacompra(1750.00);
    bean.setCantidadguiacompra("60");
    
    int salida = servicio.actualizar(bean);
    if (salida > 0) {
        System.out.println("Purchase order updated");
    }
}
```

### Retrieving Purchase Order for Validation

Before registering Bienes, validate the purchase order exists:

```java
int codigoCompra = Integer.parseInt(request.getParameter("codigocompra"));

    Bienes bien = new Bienes();
    bien.setCodigoOrdendeCompra(codigoCompra);

}
```

## Error Handling

The service layer returns integer values for CUD operations:

- **Success:** Return value `> 0` indicates the number of affected rows
- **Failure:** Return value `<= 0` indicates operation failed

### Common Error Scenarios

1. **Invalid Provider:** Registering a purchase order with a non-existent provider name may cause issues

2. **Referenced by Bienes:** Cannot delete a purchase order that is referenced by existing Bienes records (foreign key constraint)

3. **Invalid Data Format:**
   - `cantidadguiacompra` is stored as String but should contain numeric values
   - `precioguiacompra` must be a valid Double
   - `fecha_compra` must be a valid SQL Date

4. **Entity Not Found:** Updating or deleting a non-existent purchase order returns `0`

### Validation Best Practices

```java
// Before deleting, check for dependencies
GuiayOrdendeCompra orden = servicio.buscarPorId(cod);
if (orden != null) {
    // Check if Bienes reference this order
    List<Bienes> bienes = bienesService.listarBienesporCodigodeComptra(cod);
    
    if (bienes.isEmpty()) {
        // Safe to delete
        int result = servicio.eliminarPorId(cod);
    } else {
        // Cannot delete - has dependencies
        System.out.println("Cannot delete: Purchase order has associated goods");
    }
}
```

## Business Rules

### Purchase Order Lifecycle

1. **Creation:** Purchase order is created with provider and item details
2. **Good Association:** Bienes records reference the purchase order code
3. **Validation:** System validates purchase order existence before creating Bienes
4. **Deletion Constraint:** Cannot delete purchase orders with associated goods

### Data Relationships

### Field Considerations

- **nom_provee:** Should match an existing provider name in the Proveedor table
- **cantidadguiacompra:** Stored as String for flexibility but typically contains numeric quantity
- **precioguiacompra:** Total price for the purchase order
- **fecha_compra:** Date when the purchase order was created
