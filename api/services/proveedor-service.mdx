## Overview

**Package:** `com.bienes.services`

**Full Class Name:** `com.bienes.services.ProveedorServices`

The `ProveedorServices` class provides business logic for managing providers/suppliers in the system. It handles all CRUD operations for the `Proveedor` entity, acting as an intermediary between servlets and the data access layer.

### Dependencies

- **DAO Factory:** `DAOFactory.getDAOFactory(1)` - MySQL implementation
- **DAO Interface:** `ProveedorDAO`
- **Entity:** `com.bienes.entidad.Proveedor`

## Error Handling

The service layer returns integer values for CUD operations (Create, Update, Delete):

- **Success:** Return value `> 0` indicates the number of affected rows
- **Failure:** Return value `<= 0` indicates operation failed

### Common Error Scenarios

1. **Duplicate Provider:** Attempting to register a provider with duplicate information may fail depending on database constraints

2. **Entity Not Found:** When updating or deleting a non-existent Proveedor, the operation returns `0`

3. **Foreign Key Constraints:** Cannot delete a provider that is referenced by Bienes or other entities

4. **Invalid Data:** Missing required fields or invalid format (e.g., phone number) may cause failures

### Servlet-Level Error Handling

Servlets validate input and check return values:

```java
if (Integer.parseInt(id) == 0) {
    // New registration
    int salida = servicio1.registrarPro(bean);
    if (salida > 0) {
        request.setAttribute("MENSAJE", "Proveedor registrado correctamente");
    } else {
        request.setAttribute("MENSAJE", "Error en el registro del Proveedor");
    }
} else {
    // Update existing
    bean.setId_prove(Integer.parseInt(id));
    int salida = servicio1.actualizarPro(bean);
    // ... error handling
}
```

## Business Rules

### Provider Status

Providers have an `estado_prove` field that tracks their status:
- Active providers can be associated with new purchase orders
- The system should validate provider status before creating transactions

### Provider Information

All provider fields are important for:
- **Contact Management:** `telf_prove` for communication
- **Location Tracking:** `dist_prove` for logistics
- **Identification:** `nom_prove` for display in forms and reports
