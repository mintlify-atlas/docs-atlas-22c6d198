## Overview

The `OperadorDAO` interface and its MySQL implementation provide CRUD operations for managing **operadores** (operators/personnel) in the system. Operators are staff members or personnel involved in operations, logistics, or asset management.

### Interface

**Package**: `com.bienes.interfaces`

```java OperadorDAO.java
public interface OperadorDAO {
    public int saveOpe(Operador bean);
    public int updateOpe(Operador bean);
    public int deleteOpe(int codi);
    public Operador findByIdOpe(int codi);
    public List<Operador> listAll();
}
```

### Implementation

**Package**: `com.bienes.dao`  
**Class**: `MySqlOperadorDAO`  
**Location**: `com/bienes/dao/MySqlOperadorDAO.java`

## Database Schema

### Table: `operador`

| Column | Type | Description |
|--------|------|-------------|
| `id_ope` | INT (PK, AUTO_INCREMENT) | Unique operator identifier |
| `nom_ope` | VARCHAR | Operator name |
| `dist_ope` | VARCHAR | District/location |
| `cel_ope` | INT | Cell phone number |
| `estado_ope` | VARCHAR | Status (active/inactive) |

## Usage Examples

### Creating a New Operator

```java
// Obtain DAO instance
DAOFactory factory = DAOFactory.getDAOFactory(DAOFactory.MYSQL);
OperadorDAO dao = factory.getOperadorDAO();

// Create operator object
Operador operador = new Operador();
operador.setNom_ope("Juan Carlos Pérez");
operador.setDist_ope("San Isidro");
operador.setCel_ope(987654321);
operador.setEstado_ope("Activo");

// Save to database
int result = dao.saveOpe(operador);
if(result > 0) {
    System.out.println("Operator created successfully");
}
```

### Listing Active Operators

```java
OperadorDAO dao = factory.getOperadorDAO();
List<Operador> operadores = dao.listAll();

System.out.println("Active Operators:");
for(Operador op : operadores) {
    if("Activo".equals(op.getEstado_ope())) {
        System.out.println(op.getNom_ope() + " - " + op.getDist_ope());
        System.out.println("Contact: " + op.getCel_ope());
        System.out.println("---");
    }
}
```

### Updating Operator Contact

```java
OperadorDAO dao = factory.getOperadorDAO();
Operador operador = dao.findByIdOpe(25);

if(operador != null) {
    operador.setCel_ope(999888777);
    operador.setDist_ope("Miraflores");
    int result = dao.updateOpe(operador);
    if(result > 0) {
        System.out.println("Operator contact updated");
    }
}
```

### Soft Delete (Deactivate Operator)

```java
OperadorDAO dao = factory.getOperadorDAO();
Operador operador = dao.findByIdOpe(30);

if(operador != null) {
    operador.setEstado_ope("Inactivo");

    }
}
```

## Data Considerations

### Phone Number Storage

<Warning>
The `cel_ope` field uses `INT` type, which has limitations:
- Maximum value: 2,147,483,647
- Cannot store international format (+51)
- Consider using VARCHAR for flexibility
</Warning>

Example validation:

```java
if(operador.getCel_ope() < 900000000 || operador.getCel_ope() > 999999999) {
    System.err.println("Invalid Peruvian cell phone number");
}
```

### Status Values

Common status values used in `estado_ope`:

| Status | Description |
|--------|-------------|
| `Activo` | Active operator, currently working |
| `Inactivo` | Inactive/terminated |
| `Vacaciones` | On vacation |
| `Licencia` | On leave |
| `Suspendido` | Temporarily suspended |

<Note>
Status values are stored as strings. Consider using an enum or lookup table for better data integrity and validation.
</Note>
