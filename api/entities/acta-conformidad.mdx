## Overview

The `ActaConformidad` entity represents conformity certificates (actas de conformidad) issued when goods are received and verified. This entity documents that delivered items match the purchase order specifications and are in acceptable condition.

**Package:** `com.bienes.entidad`

## Fields Reference

<ParamField path="codigo_acta" type="int">
  Unique identifier for the conformity certificate (primary key)
</ParamField>

<ParamField path="descrip_bien" type="String">
  Description of the received goods/assets
</ParamField>

<ParamField path="cantidad_bien" type="int">
  Quantity of goods verified in the certificate
</ParamField>

<ParamField path="precio_bien" type="double">
  Unit price of the goods (for verification against purchase order)
</ParamField>

<ParamField path="nom_provee" type="String">
  Name of the supplier who delivered the goods
</ParamField>

<ParamField path="estado" type="String">
  Status of the conformity certificate (e.g., "APROBADO", "PENDIENTE", "RECHAZADO")
</ParamField>

## Usage Example

### Creating Conformity Certificate

```java
// Create certificate upon goods receipt
ActaConformidad acta = new ActaConformidad();
acta.setDescrip_bien("Laptop Dell Latitude 5420");
acta.setCantidad_bien(10);
acta.setPrecio_bien(1250.00);
acta.setNom_provee("Tech Suppliers SAC");
acta.setEstado("APROBADO");

// Save to database
// Note: DAO implementation not included in current source
```

### Verification Workflow

```java
// Verify received goods match purchase order
if (acta.getCantidad_bien() == ordenCompra.getCantidad() &&
    acta.getPrecio_bien() == ordenCompra.getPrecio()) {
    acta.setEstado("APROBADO");
} else {
    acta.setEstado("PENDIENTE");
    // Flag for review
}
```

## Business Process

<Steps>
  <Step title="Goods Arrival">
    Supplier delivers goods according to purchase order
  </Step>
  
  <Step title="Quality Inspection">
    Receiving department verifies quantity, quality, and specifications
  </Step>
  
  <Step title="Certificate Creation">
    ActaConformidad is created documenting the inspection results
  </Step>
  
  <Step title="Approval">
    If goods meet requirements, status set to "APROBADO"
  </Step>
  
  <Step title="Asset Registration">
    Approved goods are registered as Bienes in the inventory system
  </Step>
</Steps>

## Status Values

<Accordion title="Certificate Status Codes">

| Status | Description | Action |
|--------|-------------|--------|
| PENDIENTE | Awaiting inspection | Schedule verification |
| APROBADO | Inspection passed | Register as assets |
| RECHAZADO | Failed inspection | Return to supplier |
| PARCIAL | Partial acceptance | Process accepted quantity |

</Accordion>
