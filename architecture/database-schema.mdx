## Database Overview

The application uses a MySQL database named `gestionbienes` (Asset Management). The schema is designed to manage assets, suppliers, operators, purchase orders, and SUNARP property registrations.

<Info>
Database schema is inferred from SQL queries in DAO classes and entity field mappings.
</Info>

<Note>

</Note>

## Relationships

### Foreign Key Constraints

<CardGroup cols={2}>
  <Card title="bienes → grr-ordendecompra" icon="link">
    **Relationship:** Many-to-One
    
    **Foreign Key:** `bienes.codigo_ordencompra`
    
    **References:** `grr-ordendecompra.codigo`
    
    Multiple assets can belong to one purchase order.
  </Card>
  
  <Card title="menu → acceso" icon="link">
    **Relationship:** Many-to-One
    
    **Foreign Key:** `menu.id_acceso`
    
    **References:** `acceso.id_acceso`
    
    Multiple menu items can be assigned to one user.
  </Card>
</CardGroup>

### Implicit Relationships

<Info>
The following relationships exist through shared field values but are not enforced by database constraints:
</Info>

- **bienes.nom_provee** references supplier names from `proveedor.nom_prove`
- **grr-ordendecompra.proveedor** references supplier names from `proveedor.nom_prove`
- **inscripcion_sunarp.nom_provee** references supplier names from `proveedor.nom_prove`

<Note>
Consider adding proper foreign key constraints to enforce referential integrity for supplier relationships.
</Note>

## Database Connection

The application uses JDBC for database connectivity:

```java
// MySqlConexion.java:9-27
public static Connection getConectar() {
  Connection cn = null;
  try {
    Class.forName("com.mysql.cj.jdbc.Driver");
    String url = "jdbc:mysql://localhost:3306/gestionbienes?serverTimezone=UTC";
    String user = "root";
    String pass = "mysql";
    cn = DriverManager.getConnection(url, user, pass);
  } catch (ClassNotFoundException | SQLException e) {
    e.printStackTrace();
  }
  return cn;
}
```

**Configuration:**
- **Database:** `gestionbienes`
- **Host:** `localhost:3306`
- **Driver:** `com.mysql.cj.jdbc.Driver`
- **Timezone:** UTC

## Schema Evolution

When modifying the schema:

1. Update the corresponding entity class in `com.bienes.entidad`
2. Modify SQL queries in the relevant DAO class
3. Update the interface if method signatures change
4. Test all CRUD operations thoroughly
5. Consider database migration scripts for production

<Note>
Always backup the database before applying schema changes in production environments.
</Note>
