This guide walks through setting up the MySQL database for the asset management application.

## Prerequisites

- MySQL Server 5.7 or higher (MySQL 8.0+ recommended)
- MySQL client or MySQL Workbench
- Sufficient privileges to create databases and users
- At least 100MB of free disk space

## MySQL Installation

<Steps>
  <Step title="Download MySQL">
    Download MySQL Community Server from [mysql.com](https://dev.mysql.com/downloads/mysql/)
    
    Choose the version appropriate for your operating system:
    - **Windows:** MySQL Installer
    - **Linux:** Use package manager
    - **macOS:** DMG installer or Homebrew
  </Step>
  
  <Step title="Install MySQL Server">
    <CodeGroup>
    
    ```bash Ubuntu/Debian
    sudo apt update
    sudo apt install mysql-server
    sudo systemctl start mysql
    sudo systemctl enable mysql
    ```
    
    ```bash CentOS/RHEL
    sudo yum install mysql-server
    sudo systemctl start mysqld
    sudo systemctl enable mysqld
    ```
    
    ```bash macOS (Homebrew)
    brew install mysql
    brew services start mysql
    ```
    
    ```powershell Windows
    # Run the MySQL Installer and follow the wizard
    # Select "Server Only" or "Full" installation
    ```
    
    </CodeGroup>
  </Step>
  
  <Step title="Secure MySQL Installation">
    Run the security script to set root password and remove test databases:
    
    ```bash
    sudo mysql_secure_installation
    ```
    
    Answer the prompts:
    - Set root password: **Yes** (use a strong password)
    - Remove anonymous users: **Yes**
    - Disallow root login remotely: **Yes** (for production)
    - Remove test database: **Yes**
    - Reload privilege tables: **Yes**
  </Step>
  
  <Step title="Verify Installation">
    ```bash
    mysql --version
    ```
    
    Expected output: `mysql  Ver 8.0.x for ...`
  </Step>
</Steps>

## Database Creation

<Steps>
  <Step title="Connect to MySQL">
    ```bash
    mysql -u root -p
    ```
    
    Enter the root password when prompted.
  </Step>
  
  <Step title="Create Database">
    ```sql
    CREATE DATABASE gestionbienes
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;
    ```
    
    <Info>
      Using `utf8mb4` ensures full Unicode support including emojis and special characters.
    </Info>
  </Step>
  
  <Step title="Select Database">
    ```sql
    USE gestionbienes;
    ```
  </Step>
</Steps>

## Table Schemas

Create the following tables based on the application's data access layer.

### User Authentication Tables

<Accordion title="acceso - User Access Table">
  Stores user credentials for authentication.
  
  ```sql
  CREATE TABLE acceso (
    id_acceso INT AUTO_INCREMENT PRIMARY KEY,
    login_acceso VARCHAR(50) NOT NULL UNIQUE,
    pass_acceso VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_login (login_acceso)
  ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
  ```
  
  **Columns:**
  - `id_acceso`: Primary key, auto-incremented
  - `login_acceso`: Username for login (unique)
  - `pass_acceso`: User password (should be hashed in production)
  - `created_at`: Account creation timestamp
</Accordion>

<Accordion title="menu - User Menu Options">
  Stores menu options available to each user.
  
  ```sql
  CREATE TABLE menu (
    id_menu INT AUTO_INCREMENT PRIMARY KEY,
    id_acceso INT NOT NULL,
    descrip_menu VARCHAR(100) NOT NULL,
    url_menu VARCHAR(200) NOT NULL,
    icon_menu VARCHAR(50),
    orden INT DEFAULT 0,
    FOREIGN KEY (id_acceso) REFERENCES acceso(id_acceso) ON DELETE CASCADE,
    INDEX idx_acceso (id_acceso)
  ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
  ```
  
  **Columns:**
  - `id_menu`: Primary key
  - `id_acceso`: Foreign key to acceso table
  - `descrip_menu`: Menu item description
  - `url_menu`: Target URL for menu item
  - `icon_menu`: Icon identifier for UI
  - `orden`: Display order
</Accordion>

## Database User Permissions

<Steps>
  <Step title="Create Application User">
    Create a dedicated database user for the application:
    
    ```sql
    CREATE USER 'gestionbienes_app'@'localhost' 
    IDENTIFIED BY 'secure_password_here';
    ```
    
    For remote connections:
    ```sql
    CREATE USER 'gestionbienes_app'@'%' 
    IDENTIFIED BY 'secure_password_here';
    ```
  </Step>
  
  <Step title="Grant Necessary Privileges">
    Grant only required permissions:
    
    ```sql
    -- For development (full access)
    GRANT ALL PRIVILEGES ON gestionbienes.* 
    TO 'gestionbienes_app'@'localhost';
    
    -- For production (restricted access)
    GRANT SELECT, INSERT, UPDATE, DELETE ON gestionbienes.* 
    TO 'gestionbienes_app'@'localhost';
    
    FLUSH PRIVILEGES;
    ```
  </Step>
  
  <Step title="Verify Permissions">
    ```sql
    SHOW GRANTS FOR 'gestionbienes_app'@'localhost';
    ```
  </Step>
  
  <Step title="Update Application Configuration">
    Update `MySqlConexion.java` with the new user:
    
    ```java
    user="gestionbienes_app";
    pass="secure_password_here";
    ```
  </Step>
</Steps>

<Info>
  The principle of least privilege: Grant only the minimum permissions required for the application to function.
</Info>

## Backup and Recovery

### Database Backup

<Steps>
  <Step title="Full Database Backup">
    ```bash
    mysqldump -u root -p gestionbienes > gestionbienes_backup_$(date +%Y%m%d).sql
    ```
  </Step>
  
  <Step title="Compressed Backup">
    ```bash
    mysqldump -u root -p gestionbienes | gzip > gestionbienes_backup_$(date +%Y%m%d).sql.gz
    ```
  </Step>
  
  <Step title="Backup Specific Tables">
    ```bash
    mysqldump -u root -p gestionbienes acceso menu > users_backup.sql
    ```
  </Step>
</Steps>

### Database Recovery

<Steps>
  <Step title="Restore from Backup">
    ```bash
    mysql -u root -p gestionbienes < gestionbienes_backup_20260304.sql
    ```
  </Step>
  
  <Step title="Restore from Compressed Backup">
    ```bash
    gunzip < gestionbienes_backup_20260304.sql.gz | mysql -u root -p gestionbienes
    ```
  </Step>
</Steps>

### Automated Backup Script

Create a cron job for daily backups:

```bash
#!/bin/bash
# backup_db.sh

BACKUP_DIR="/var/backups/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="gestionbienes"

mkdir -p $BACKUP_DIR
mysqldump -u backup_user -p'backup_password' $DB_NAME | gzip > $BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz

# Keep only last 30 days of backups
find $BACKUP_DIR -name "${DB_NAME}_*.sql.gz" -mtime +30 -delete
```

Schedule with crontab:
```bash
# Run daily at 2 AM
0 2 * * * /path/to/backup_db.sh
```

## Performance Optimization

### Indexing Strategy

The schema includes indexes on frequently queried columns:

- `acceso.login_acceso` - Fast user authentication
- `proveedor.nom_prove` - Quick provider lookups
- `bienes.fecha_llegada` - Date range queries
- `bienes.codigo_ordencompra` - Join optimization

### Connection Pooling

For production, implement connection pooling instead of creating connections per request. See [configuration guide](/deployment/configuration) for details.

## Troubleshooting

<Accordion title="ERROR 1045: Access denied for user">
  **Cause:** Incorrect username or password
  
  **Solution:**
  1. Verify user exists: `SELECT User, Host FROM mysql.user;`
  2. Reset password: `ALTER USER 'username'@'localhost' IDENTIFIED BY 'newpassword';`
  3. Update application configuration
</Accordion>

<Accordion title="ERROR 1049: Unknown database">
  **Cause:** Database not created
  
  **Solution:**
  ```sql
  SHOW DATABASES;
  CREATE DATABASE IF NOT EXISTS gestionbienes;
  ```
</Accordion>

<Accordion title="ERROR 1215: Cannot add foreign key constraint">
  **Cause:** Referenced table doesn't exist or column types don't match
  
  **Solution:**
  1. Create tables in correct order (parent tables first)
  2. Verify column types match exactly
  3. Ensure referenced columns have indexes
</Accordion>

<Accordion title="Slow query performance">
  **Cause:** Missing indexes or inefficient queries
  
  **Solution:**
  1. Enable slow query log: `SET GLOBAL slow_query_log = 'ON';`
  2. Analyze queries: `EXPLAIN SELECT ...`
  3. Add indexes to frequently queried columns
  4. Implement connection pooling
</Accordion>
